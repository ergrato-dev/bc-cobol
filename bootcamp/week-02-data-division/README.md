# Semana 02: DATA DIVISION

## 🎯 Objetivos

Al finalizar esta semana, el estudiante será capaz de:

- Diseñar layouts de datos con PICTURE clause para tipos alfanuméricos y numéricos
- Usar USAGE clause para optimizar almacenamiento (DISPLAY, COMP, COMP-3)
- Inicializar variables con VALUE clause
- Estructurar datos jerárquicamente con niveles 01-49
- Diferenciar entre FILE SECTION y WORKING-STORAGE SECTION
- Declarar variables elementales (PIC) y de grupo

## 📚 Contenido

### 1-teoria/

- **picture-clause.md**: Símbolos de edición (9, X, A, V, S, signo, decimal)
- **usage-clause.md**: DISPLAY, COMP, COMP-3, BINARY, PACKED-DECIMAL
- **value-clause.md**: Inicialización, literales, figurativas (ZEROS, SPACES)
- **niveles-01-49.md**: Jerarquía de datos, grupos, elementales, 77, 88
- **sections-working-storage.md**: WORKING-STORAGE, FILE SECTION, LINKAGE SECTION

### 2-practicas/

- **layout-cliente.cbl**: Diseño de registro de cliente bancario
- **jerarquia-factura.cbl**: Registro de factura con múltiples niveles
- **inicializacion.cbl**: Uso de VALUE y constantes figurativas

### 3-proyecto/

- **starter/ficha-empleado.cbl**: Sistema de ficha de empleado con datos estructurados (nombre, salario, fecha ingreso, departamento)

## ⏱️ Distribución (10h)

- Teoría: 3h | Prácticas: 4h | Proyecto: 3h

## 📌 Entregables

- [ ] Layout de cliente bancario con PICTURE correcto
- [ ] Jerarquía de factura con al menos 3 niveles
- [ ] Proyecto de ficha de empleado completado
