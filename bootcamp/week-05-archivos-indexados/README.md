# Semana 05: Archivos Indexados

## 🎯 Objetivos

Al finalizar esta semana, el estudiante será capaz de:

- Configurar archivos con ORGANIZATION IS INDEXED
- Implementar ACCESS MODE SEQUENTIAL, RANDOM y DYNAMIC
- Usar RECORD KEY y ALTERNATE RECORD KEY
- Realizar operaciones START, DELETE, REWRITE
- Manejar INVALID KEY y FILE STATUS en archivos indexados
- Diseñar índices primarios y secundarios para acceso eficiente

## 📚 Contenido

### 1-teoria/

- **organizacion-indexada.md**: INDEXED, estructura de índice, registros
- **record-key.md**: PRIMARY KEY, ALTERNATE KEY, duplicados
- **modos-acceso.md**: SEQUENTIAL, RANDOM, DYNAMIC
- **verbos-indexados.md**: START, READ NEXT, DELETE, REWRITE
- **comparativa-archivos.md**: Secuencial vs Indexado vs Relativo

### 2-practicas/

- **crear-indexado.cbl**: Crear archivo indexado desde datos secuenciales
- **consulta-aleatoria.cbl**: Acceso RANDOM por clave
- **mantenimiento.cbl**: Altas, bajas y cambios en archivo indexado

### 3-proyecto/

- **starter/catalogo-productos.cbl**: Sistema CRUD de catálogo de productos bancarios con archivo indexado
