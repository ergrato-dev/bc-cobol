# Semana 10: SORT y MERGE

## 🎯 Objetivos

Al finalizar esta semana, el estudiante será capaz de:

- Implementar SORT con USING y GIVING
- Configurar SD (Sort Description) en FILE SECTION
- Usar INPUT PROCEDURE y OUTPUT PROCEDURE con SORT
- Fusionar archivos con MERGE
- Manejar archivos temporales de ordenamiento
- Procesar grandes volúmenes de datos con SORT eficiente

## 📚 Contenido

### 1-teoria/

- **sort-statement.md**: SORT, ASCENDING/DESCENDING KEY, USING, GIVING
- **sd-file.md**: SD en FILE SECTION, estructura de archivo de sort
- **input-output-procedure.md**: Procesamiento antes/después del sort
- **merge.md**: MERGE, múltiples archivos de entrada, archivo fusionado
- **rendimiento-sort.md**: Memoria, archivos temporales, SORTWK

### 2-practicas/

- **ordenar-clientes.cbl**: SORT simple por nombre
- **ordenar-con-filtro.cbl**: INPUT PROCEDURE filtrar, SORT, OUTPUT PROCEDURE formatear
- **fusion-archivos.cbl**: MERGE de archivos de transacciones de sucursales

### 3-proyecto/

- **starter/consolidacion-diaria.cbl**: Consolidar transacciones de múltiples sucursales, ordenar por cuenta, generar reporte maestro

- Teoría: 3h | Prácticas: 4h | Proyecto: 3h
